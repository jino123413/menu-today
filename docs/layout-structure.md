# 오늘 메뉴 추천 레이아웃 구조도

## 앱 화면 구조
- 홈/추천 탭(입력 + 추천 실행)
- 기록 탭
- 즐겨찾기 탭
- 통계 탭

```text
App
├─ DeviceViewport
├─ app-header
│  ├─ title (오늘 메뉴 추천)
│  └─ attempt-chip (오늘 시도 횟수)
├─ tab-bar
│  ├─ 추천
│  ├─ 기록
│  ├─ 즐겨찾기
│  └─ 통계
└─ app-content
   ├─ 탭: 추천
   │  ├─ 입력 조건 카드
   │  │  ├─ 식사시간 / 인원 / 매운맛 / 예산 / 식단성향
   │  │  ├─ 최대조리시간 / 피하고 싶은 재료
   │  │  └─ 추천 버튼 / 초기화 버튼
   │  └─ 오늘 추천 결과 카드
   │     ├─ 추천 메뉴 카드
   │     ├─ 추천 사유 목록
   │     └─ 대안 메뉴 카드 리스트
   ├─ 탭: 기록
   │  ├─ 기록 리스트
   │  ├─ 기록 상세(점수, 사유, 시간)
   │  └─ 기록 삭제 버튼
   ├─ 탭: 즐겨찾기
   │  ├─ 즐겨찾기 메뉴 카드 리스트
   │  └─ 하트 토글
   └─ 탭: 통계
      ├─ 핵심 지표(총 횟수/평균 인원/평균 점수)
      ├─ 최근 메뉴 목록
      └─ 식사시간/요리 종류 Top 뷰
```

## 상태/데이터 흐름
- 추천 조건은 입력값으로 시그니처를 만들고 일일 상태(`DailyState`)와 매칭
- 추천 시도 시 중복 방지 ID를 사용해 같은 메뉴 과다 노출 제한
- 기록은 로컬 스토리지에 `menu-today-history`로 저장
- 즐겨찾기는 `menu-today-favorites`로 저장
